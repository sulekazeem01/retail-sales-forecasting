HOW TO INTERPRET DF.INFO()¶

df.info() gives you a summary of your entire dataset, showing:

  1. Total number of rows This tells you how many records (observations) your dataset contains.
  1. Column names A list of ALL columns in the DataFrame.
  1. Non-null counts Shows how many non-missing (not NaN) values each column has. If the number is less than the total rows, the column contains missing values. Example: Age 950 non-null Gender 1000 non-null → Age has 50 missing values.
  1. Data types (dtypes) Each column’s data type, such as: int64 → whole numbers float64 → decimal numbers object → text or mixed values datetime64 → dates bool → True/False This helps you know whether your columns are in the correct format.
  1. Memory usage Shows how much RAM your DataFrame is using.

Distribution of Total Spending¶

In [6]:
sns.histplot(df['TotalSpending'], bins=30, kde=True)
plt.title("Distribution of Total Spending")
plt.show()

Brief Interpretation of the Distribution of Total Spending¶

This histogram with a KDE curve shows that Total Spending is right-skewed (positively skewed).

What this means:

Most customers spend a smaller amount, clustering between 0 and 1,000.

As spending increases, the number of customers decreases.

A long right tail shows there are some high-spending customers (2,000–4,500), but they are much fewer.

This indicates a minority of customers contribute the highest revenue — a common pattern in retail (Pareto principle: 20% of customers drive 80% of sales).

⭐ Why This Matters (in data analysis or interviews)

Most revenue may come from a small group of high spenders → useful for segmentation.

Right-skewed distributions often benefit from transformation (e.g., log transform) before modeling.

Shows high variance in customer purchasing behavior.

# 1) One-hot encode before the split¶

To encode categorical features (Gender, ProductCategory) to numeric form for regression.

One-hot encode categorical variables

In [10]:
X = pd.get_dummies(X, drop_first=True).astype(float)

Brief Interpretation¶

  1. One-hot Encode the Categorical Columns X = pd.get_dummies(X, drop_first=True).astype(float)

This line converts categorical variables (Gender, ProductCategory) into numeric dummy variables because machine-learning models can’t read text.

pd.get_dummies() Turns each category into 0/1 columns.

drop_first=True Prevents multicollinearity by dropping one dummy from each category group.

astype(float) Converts everything into numeric so the model can train.

🎯 Why this is important

You’re preparing the dataset for machine learning:

X = input variables

y = what you’re predicting

get_dummies = convert categories to numbers

drop_first = avoid dummy trap

In [13]:
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=70)

Brief Interpretation¶

This line splits your dataset into training and testing sets.

✔ What each part means:

X = all your input features

y = your target variable (what you're predicting)

✔ After the split:

X_train → features used to train the model

X_test → features the model will be tested on

y_train → labels the model learns from

y_test → actual labels used to check prediction accuracy

✔ Key parameters:

test_size=0.3 → 30% of the data becomes the test set, 70% is for training.

random_state=70 → Ensures the split is reproducible. (You get the same split every time.)

⭐ In simple terms:

👉 “Split the dataset so the model learns from 70% of the data and is tested on the remaining 30%, and keep the split repeatable.”
